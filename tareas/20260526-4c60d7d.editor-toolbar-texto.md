# Spec: Editor Toolbar - Texto Enlazado y Estados Mixtos

## Objective
Arreglar el comportamiento de la floating toolbar (`EditorFloatingToolbar.vue`) y del editor (`RichTextEditor.vue`) en cajas de texto enlazable (`linkedText`) para que:
1. Distinga correctamente 3 estados: caja seleccionada, editando texto, texto seleccionado
2. Refleje estados mixtos (varias fuentes, tamaños, colores) en la toolbar
3. Preserve el formato al pegar texto enriquecido y salir del modo edición
4. Genere `TEXTO_EDITOR.md` documentando todas las reglas

## Comportamiento Detallado

### Estado A: Caja seleccionada (click simple, NO editando)

| Control | Comportamiento |
|---------|---------------|
| Font-family | Muestra nombre de fuente si todo el texto usa la misma. Muestra "Varios" si hay múltiples. **Activo**: al cambiar, afecta TODO el texto |
| Font-size | Muestra el valor si todo usa el mismo tamaño. Input en blanco si hay variedad. **Activo**: al cambiar, afecta TODO el texto |
| Color | Muestra el color sólido si todo usa el mismo. Muestra **gradiente** si hay variedad. **Activo**: al cambiar, afecta TODO el texto |
| Bold | Botón **gris** si hay mixed state (parte bold, parte no). Si todo bold → activo. Si nada bold → inactivo. **Activo**: al clickear, aplica bold a TODO el texto o lo quita de TODO |
| Italic | Mismo comportamiento que bold (gris si mixed) |
| Underline | Mismo comportamiento que bold (gris si mixed) |
| Uppercase | Mismo comportamiento (gris si mixed) |
| Alignment | Muestra icono actual si todo texto tiene misma alineación. Muestra icono de **raya** si mixed. **Activo**: al cambiar, afecta TODO el texto |
| Spacing | Mismo comportamiento (gris/variado si mixed) |
| Lista | **Activo**: aplica/quita lista de TODO el texto (bullet/ordered/none) |
| Effects | **Siempre activo**: aplica a nivel de elemento (no de texto) |
| Opacity | **Siempre activo**: aplica a nivel de elemento (no de texto) |

### Estado B: Editando texto (cursor activo dentro del editor)

- Los controles reflejan las **propiedades del párrafo actual** donde está el cursor
- Si el cursor está al inicio sin contenido seleccionado, se muestran los valores por defecto del párrafo
- **Effects y Opacity** no se muestran o aparecen deshabilitados (son de elemento, no de texto)

### Estado C: Texto seleccionado (rango de texto marcado)

- Bold/Italic/Underline/Color/Font-size/Font-family se aplican como **marks** (a nivel de carácter/selección), no como atributos de párrafo
- Alignment/Spacing/Lista se aplican a nivel de párrafo (si la selección abarca múltiples párrafos, afecta todos)

### Preservación de formato al pegar

- El `handlePaste` en `RichTextEditor.vue` debe ser corregido para que el `confirm` de "conservar formato" aparezca correctamente (actualmente no se muestra)
- `mergeParagraphStylesIntoHtml` debe **preservar todos los estilos inline** del HTML pegado:
  - Propiedades existentes en el HTML pegado tienen **prioridad**
  - Solo se completan propiedades faltantes con los valores del layout
- Al salir del modo edición, el texto debe verse **idéntico** a como se veía dentro del editor

### Cómo detectar mixed state

Para determinar si un estilo es uniforme o variado en todo el texto de la caja:
1. Recorrer todos los párrafos del HTML o del `paragraphStyles`
2. Comparar cada propiedad (fontFamily, fontSize, color, fontWeight, italic, underline, uppercase, textAlign, letterSpacing, lineHeight)
3. Si algún párrafo difiere → mixed state
4. Los botones bold/italic/underline tienen 3 estados visuales:
   - `btn-accent` (activo) → todo el texto tiene esa propiedad
   - `btn-outline` (inactivo) → ningún texto tiene esa propiedad
   - Clase gris (`btn-ghost` o similar) → mixed state

## Success Criteria
1. Al pegar texto enriquecido desde una página web en una caja linkedText, el formato se preserva **íntegramente** al salir del modo edición (font-size, font-family, color, bold, italic, listas)
2. La toolbar muestra correctamente "Varios" / gradiente / input vacío / botón gris cuando hay mixed state
3. Desde el estado A (caja seleccionada), al cambiar fuente/tamaño/color/bold/italic/etc se afecta **todo** el texto de la caja
4. Desde el estado C (selección de texto), bold/italic/color/font-size se aplican solo a la selección (marks)
5. `TEXTO_EDITOR.md` documenta todas las reglas

## Boundaries
- **Always do:** Preservar inline styles del HTML pegado, mostrar mixed states correctamente
- **Ask first:** Cambiar el sistema de fragmentación (`useLinkedTextBoxSystem.js`), modificar lógica de enlaces
- **Never do:** Modificar el sistema de redistribución de texto entre cajas enlazadas, romper la lógica de linkedTextNext/Prev
